# Contributing

Thanks for your interest in contributing to this project.

## Before You Open a Pull Request

All contributions are subject to the Contributor License Agreement in `CLA.md`.
Repository maintainers can use `.github/cla-setup.md` to configure and maintain the GitHub-side CLA signing workflow.

By signing the CLA:

- you keep ownership of your own contributions;
- you give Eric Blois broad rights to use, modify, distribute, commercialize, and relicense those contributions; and
- you agree that you are not entitled to royalties, revenue sharing, or other compensation from use of those contributions.

If you are contributing on behalf of an employer or another entity, make sure you have the authority to do so before opening a pull request.

## Pull Request Process

1. Make your changes in a focused branch.
2. Open a pull request with a clear description of what changed.
3. Complete the CLA signing flow if the repository bot asks you to sign.
4. Address review feedback and keep the branch up to date until the pull request is ready to merge.
